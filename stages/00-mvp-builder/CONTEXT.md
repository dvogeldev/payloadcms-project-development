# Stage 00: MVP Development with Online Builder or LLM (Optional Precursor)

## Inputs
| Source | File/Location | Section/Scope | Why |
|--------|--------------|---------------|-----|
| User | (conversation) | Basic concept, rough requirements, design inspiration | Initial project idea to validate |

## Process
1. Develop a minimum viable product (MVP) using online builders like lovable.dev, v0.dev, bolt.new, antigravity, or plain LLM prompts that generate project folders
2. Use the MVP to validate concepts, gather feedback, and refine requirements
3. Extract design patterns, component structures, and feature priorities from the MVP

## Outputs
| Artifact | Location | Format |
|----------|----------|--------|
| MVP feedback report | `output/mvp-feedback-report.md` | Markdown document |
| Refined requirements | `output/refined-requirements.md` | Markdown document |
| Design specifications | `output/design-specifications.md` | Markdown document |
